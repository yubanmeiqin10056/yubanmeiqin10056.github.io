# 折腾博客的奇妙旅程：从Markdown渲染失败到Jekyll重构

今天真是折腾博客的一天！原本以为我的GitHub Pages博客已经搭建好了，结果发现了一个大问题——Markdown文章根本无法正确渲染！这让我踏上了一段奇妙的调试和重构之旅。

## 问题的发现

一切都要从我发现的一个奇怪现象说起。当我访问博客文章时，比如 `https://yubanmeiqin10056.github.io/post/2025-02-08极客湾.md`，页面上的内容完全不对劲。

我写的文章开头是这样的：
```markdown
#怀念
"15年前我们怀着满腔热血..."
```

但是在网页上显示的却是纯文本，连 `#` 标题符号都原样显示了出来，完全没有变成HTML的 `<h1>` 标题。这让我意识到——我的博客根本没有正确处理Markdown格式！

## 深入分析问题

经过一番检查，我发现了问题的根源：

### 1. 混乱的架构
我的博客采用了奇怪的混合架构：
- 使用Jekyll进行GitHub Pages部署
- 但同时用JavaScript动态加载和渲染内容
- 两种方案互相冲突，导致功能异常

### 2. 缺少Jekyll布局文件
Jekyll需要 `_layouts` 目录中的模板文件来正确渲染内容，但我的博客完全没有这些文件。

### 3. 文章格式不规范
我的Markdown文件缺少Jekyll需要的Front Matter（前置元数据），导致Jekyll无法识别它们。

## 解决方案：完全重构

面对这些问题，我决定彻底重构博客架构。

### 第一步：创建标准Jekyll结构

我创建了完整的Jekyll目录结构：
```
├── _layouts/
│   ├── default.html
│   └── post.html
├── _config.yml
├── index.html
└── post/
    ├── 2025-02-08极客湾.md
    ├── 2025-3-19-测试.md
    └── 某同学的文章.md
```

### 第二步：设计布局模板

我创建了两个主要的布局文件：

**default.html** - 基础布局，包含：
- 响应式设计
- 美观的样式
- 背景图片适配
- 统一的页面结构

**post.html** - 文章专用布局，包含：
- 文章标题显示
- 发布时间
- 作者信息
- 导航链接

### 第三步：规范文章格式

我为每篇文章添加了Front Matter：
```yaml
---
layout: post
title: "文章标题"
date: 2025-02-08 21:09:00 +0800
author: "御坂10056号"
---
```

这样Jekyll就能正确识别和处理文章了。

### 第四步：优化主页

我重新设计了主页，添加了：
- 欢迎区域
- 导航按钮
- 文章列表
- 仓库信息展示

## 遇到的坑

在重构过程中，我也遇到了一些有趣的问题：

### 1. Git提交权限问题
提交代码时遇到权限错误，需要配置用户信息：
```bash
git config user.name "yubanmeiqin10056"
git config user.email "389575503@qq.com"
```

### 2. GitHub Actions部署
利用GitHub Actions自动部署Jekyll网站，确保每次推送后自动更新。

### 3. 图片路径问题
修复了Markdown中图片的相对路径问题，确保图片能正确显示。
