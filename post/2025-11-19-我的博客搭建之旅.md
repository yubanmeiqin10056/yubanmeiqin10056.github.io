# 我的博客搭建之旅

作为一个初三学生，我一直想要拥有自己的博客来记录学习心得和技术笔记。今天，我终于成功搭建了自己的博客网站，这个过程充满了挑战和乐趣！

## 为什么选择Jekyll + GitHub Pages？

在选择博客平台时，我考虑了几个选项：

1. **WordPress** - 功能强大，但需要服务器和数据库
2. **Hexo** - 静态博客生成器，但配置相对复杂
3. **Jekyll + GitHub Pages** - 免费托管，支持Markdown，适合技术博客

最终我选择了Jekyll + GitHub Pages，因为：
- 完全免费，不需要自己维护服务器
- 支持Markdown写作，语法简单
- 可以自定义域名和样式
- Git版本控制，方便管理文章

## 搭建过程

### 第一步：创建GitHub仓库

首先，我创建了一个名为`yubanmeiqin10056.github.io`的仓库，这是GitHub Pages的特殊命名规则，可以直接通过`yubanmeiqin10056.github.io`访问。

### 第二步：学习Jekyll基础

Jekyll是一个静态网站生成器，它可以将Markdown文件转换成HTML网页。我学习了以下概念：

- `_config.yml` - 网站配置文件
- `_layouts` - 页面布局模板
- `_posts` - 博客文章存放目录
- `index.html` - 首页文件

### 第三步：设计页面布局

我设计了一个简洁的页面布局：

1. **首页** - 包含欢迎信息和最新文章列表
2. **文章列表页** - 展示所有博客文章
3. **文章详情页** - 显示单篇文章内容
4. **页脚** - 显示版权信息和仓库地址

### 第四步：美化样式

为了让博客看起来更美观，我添加了CSS样式：

- 响应式设计，适配手机和电脑
- 背景图片，增加视觉效果
- 卡片式布局，提升阅读体验
- 悬停效果，增加交互感

## 遇到的挑战

### 挑战1：Jekyll语法学习

刚开始我对Jekyll的Liquid语法不太熟悉，比如如何遍历文章、如何格式化日期等。通过查阅文档和不断尝试，我逐渐掌握了基本用法。

```liquid
{% for post in site.posts %}
  <h2>{{ post.title }}</h2>
  <p>{{ post.date | date: "%Y年%m月%d日" }}</p>
{% endfor %}
```

### 挑战2：CSS样式调试

在调整CSS样式时，我遇到了一些兼容性问题。特别是在手机端和桌面端的显示效果不一致。通过使用媒体查询（Media Query），我解决了这个问题。

```css
@media (max-width: 768px) {
  body {
    background-image: url('/otherfile/手机背景.png');
  }
}
```

### 挑战3：图片路径配置

在添加背景图片时，图片无法正常显示。后来发现是路径配置问题，需要使用绝对路径。
